Lab 1B

Objective

The purpose of Lab 1B is to establish communication between computer and the Artemis board through the Bluetooth stack.

Prelab

Setup

I first set up the laptop and the Artemis Nano so they could reliably communicate over BLE. On the laptop, I created a Python environment for BLE and installed the required packages from the lab codebase. On the Artemis side, I uploaded the provided ble_arduino.ino sketch, which initializes BLE, defines the GATT service/characteristics, and starts advertising.

After uploading, the Artemis printed its BLE MAC address in the Serial Monitor. This confirmed that BLE initialized correctly and the board was advertising.

Serial Monitor output confirming BLE advertising and the Artemis MAC address.

Configuration

To make sure my Python client connects to the correct board and the correct BLE interface, I updated connection.yaml. This file stores the Artemis MAC address, the BLE service UUID, and the UUIDs for each characteristic.

I generated a unique BLE service UUID in Python using uuid4() and used the same UUID in both the Arduino sketch and connection.yaml.

Generating a unique BLE service UUID in Python.

The service exposes these characteristics, which define how data moves between the laptop and the Artemis:

  • TX_CMD_STRING: laptop → Artemis (write). Used to send command strings from Python.
  • RX_STRING: Artemis → laptop (notify/read). Used to send string replies and streamed data.
  • RX_FLOAT: Artemis → laptop (notify/read). Reserved for numeric data (used in later/extended tasks).

The same service UUID and characteristic UUIDs are defined in the Arduino sketch so the Artemis advertises the expected BLE interface.

Arduino sketch: service UUID and characteristic UUID definitions.

Finally, I copied the MAC address, service UUID, and characteristic UUIDs into connection.yaml so the Python client connects to the correct device and characteristics.

connection.yaml: Artemis MAC address, service UUID, and characteristic UUIDs.

Code Base

The provided BLE codebase has two parts: an Arduino program on the Artemis (BLE peripheral) and a Python client on the laptop (BLE central). The Python client connects using the MAC address and service UUID, then sends commands by writing to TX_CMD_STRING. On the Artemis, the sketch reads the received command, parses it, and runs the corresponding handler.

After executing a command, the Artemis returns results to the laptop using BLE notifications (typically through RX_STRING). In Python, a notification handler can be registered to receive these messages asynchronously, which is especially useful for streaming data and measuring data rate.
