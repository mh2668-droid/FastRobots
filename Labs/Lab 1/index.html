Lab 1B

Objective

The purpose of Lab 1B is to establish communication between computer and the Artemis board through the Bluetooth stack.

Prelab

BLE Setup

I set up a Python environment for BLE communication and installed the required packages (including bleak). On the Artemis side, I uploaded the provided ble_arduino.ino sketch and confirmed the board was advertising by printing its BLE MAC address in the Serial Monitor.

Serial Monitor output confirming BLE advertising and the Artemis MAC address.

Configuration

To connect the Python client to my specific Artemis board, I updated connection.yaml with my Artemis MAC address (artemis_address) and a unique BLE service UUID (ble_service). I generated the UUID in Python using uuid4() and set the same service UUID in the Arduino sketch so both sides matched.

The BLE service exposes several characteristics that define how data is sent between the laptop and the Artemis:

  • TX_CMD_STRING: used by the laptop to send command strings to the Artemis (e.g., ECHO, GET_TIME_MILLIS)
  • RX_STRING: used by the Artemis to send string responses back to the laptop via notifications
  • RX_FLOAT: defined for sending numeric data (used in later or extended tasks)

Step 1: I generated a unique BLE service UUID (using uuid4()) and updated the Arduino sketch so the Artemis advertises my custom service and characteristics.

Arduino sketch: service UUID and characteristic UUID definitions.

Step 2: I copied the same service UUID and characteristic UUIDs into connection.yaml so the Python client connects to the correct BLE service on my Artemis board.

connection.yaml: Artemis MAC address, service UUID, and characteristic UUIDs.

Code Base

The provided BLE codebase has two parts: an Arduino program on the Artemis (BLE peripheral) and a Python client on the laptop (BLE central). The Python client connects using the MAC address and service UUID, then sends commands by writing to TX_CMD_STRING. On the Artemis, the sketch reads the received command, parses it, and runs the corresponding handler.

After executing a command, the Artemis returns results to the laptop using BLE notifications (typically through RX_STRING). In Python, a notification handler can be registered to receive these messages asynchronously, which is especially useful for streaming data and measuring data rate.
